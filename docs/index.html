📝 作業與課程完整工作流程
第 1 步：Fork 課程專案到你的帳號（學期做一次）

前往課程 Repo，點右上 Fork → Create fork。

之後所有作業都在你自己的 Fork 進行。
第 2 步：建立你的雲端開發環境（Codespace）
  1. 進入你 Fork 的 Repo，點綠色 <> Code
  2. 選擇 Create codespace on main
  3. 等待 1–2 分鐘，VS Code 網頁版出現即可開始
# 初次開啟後建議先載入環境
source env.sh
第 3 步：同步老師的更新（日常動作）
同步前請先確認你的作業都有 commit（用 git status 檢查）。

（A）設定上游 upstream（只做一次）

git remote add upstream https://github.com/cwbdayi638/geophysics-class-2025.git
git remote -v

（B）拉取並合併更新（每次要開始新作業前）

git fetch upstream
git merge upstream/main
git push origin main
第 4 步：在 Codespace 完成作業並儲存進度

至 /hw/hwX/ 依指示完成 學號.ipynb。

# 1) 加入所有變更
git add .

# 2) 建立版本紀錄（請修改訊息）
git commit -m "完成 HW1 第一、二題"

# 3) 推送到你的 GitHub Repo
git push
第 5 步：繳交作業（建立 Pull Request, PR）
  1. 回到你 Fork 的 Repo 頁，點 Contribute → Open pull request
  2. 確認 Base 是老師的 Repo、Head 是你的 Repo
  3. 標題建議：[HW1] 姓名 - 學號
  4. 內容需含：目的、方法、結果圖、觀察與討論
第 6 步：接收回饋與修改

老師會在 PR 直接評語/打分，如需修改，回 Codespace：

git add .
git commit -m "根據回饋修正 HW1"
git push    # 既有 PR 會自動更新
❓ 常見問題

合併衝突（Merge Conflict）怎麼辦？

通常是你和老師同時改到某檔案同一行。別緊張，這是 Git 的正常機制。

  1. VS Code 會標記衝突區塊（<<<<<<< / ======= / >>>>>>>）。
  2. 選擇保留「你的/老師的」或手動合併，刪除標記。
  3. 完成後 git add .、git commit。
  4. 仍不確定時，請儘快向老師或助教求助。

常用一鍵同步腳本

git remote -v
echo "---------------"
git fetch upstream
echo "---------------"
git merge upstream/main
echo "---------------"
git push origin main

cd /workspaces/geophysics-class-2025
